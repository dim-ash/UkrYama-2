1 <p>since version 1.6 you can finally customize the email messages used by userGroups.</p>
2 
3 <h3>Prepare your message class</h3>
4 <p>All you have to do to use this new feature is to create your email message class.<br/>
5 How can you do that? Just implement the UGMailMessage interface (located inside the UGMail.php file inside the module components directory)
6 and you are good to go.<br/>
7 The methods you'll have to implement are all self explanatory and if you have any doubt about what to do with each one of them you can take a look at
8 the already implemented classes.</p>
9 
10 <h3>What if I just want to make small changes to the default messages?</h3>
11 <p>That's even easier: just extend the class of the mail message you want to change.</p>
12 
13 <h3>How do i make userGroups aware of my new class?</h3>
14 <p>Just like Profile Extensions all you have to do is edit the module configuration located inside your application config file:</p>
15 
16 <div class="code"><code>
17  'userGroups'=&gt;array(
18  'accessCode'=&gt;'your access code, by now useless',
19  <span class="highlight">'mailMessages' =&gt; array('invitation' =&gt; 'YourClassName'),</span>
20  )
21 
22 </code></div>
23 
24 <p>Just pay attention to the following rules:</p>
25 
26 <ul>
27  <li>userGroups must be able to create an instance of your class, so if you store your class file inside some directory that
28  is not already imported by Yii, do it!
29  </li>
30  <li>if you put your class file inside a directory already imported by Yii remember to name the class file after the class name.</li>
31  <li>in this version of userGroups we are not supporting dot notation for you class name when using customized mail messages.</li>
32 </ul>
33 
34 <h3>In the above example the array key is 'invitation'. What keys do i have to use?</h3>
35 <p>Easy enough:</p>
36 
37 <dl>
38  <dt>invitation</dt>
39  <dd>the message you want to use when sending invitation emails</dd>
40  <dt>pass_reset</dt>
41  <dd>the message that will be sent when a user request a pass reset</dd>
42  <dt>activation</dt>
43  <dd>the message that will be sent to a user with the instructions to activate his own account</dd>
44 </dl>

47 <h3>All those methods have a $data argument. What's that?</h3>
48 <p>$data is just an array, ready to be used by Yii::t().<br/>
49 Still, if you don't care about localization you can use the data inside that array.<br/>
50 Here's a list of the informations stored inside that array:</p>
51 
52 <dl>
53  <dt>{email}</dt>
54  <dd>contains the user email address</dd>
55  <dt>{username}</dt>
56  <dd>contains the user username</dd>
57  <dt>{activation_code}</dt>
58  <dd>contains the user activation code. This value may be null</dd>
59  <dt>{link}</dt>
60  <dd>the link to your application user activation page</dd>
61  <dt>{full_link}</dt>
62  <dd>just like the one above, plus it's already filled with username and activation code as get parameters</dd>
63  <dt>{website}</dt>
64  <dd>your application name</dd>
65  <dt>{temporary_username}</dt>
66  <dd>a boolean value to determine if the username is temporary or not</dd>
67 </dl>

69 <h3>What if i just want to change the mail message and don't care about anything else?</h3>
70 <p>Since version 1.8 the mail messages body are stored inside some view files.<br/>
71 These view files are located inside your application view directory inside the ugmail folder.</p>
72 <p>The reason because those file are stored not inside the module folder is because in this way you won't have to
73 worry about it when updating userGroups.</p>
