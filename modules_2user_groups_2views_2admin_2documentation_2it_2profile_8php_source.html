7 <h3>Primo passo: creare la tabella del database ed il modello</h3>
8 <p>Crea semplicemente una tabella nel database, come hai sempre fatto.<br/>
9 Ricordati semplicemente di inserire una colonna chiamata <b>ug_id</b>, fai s&igrave; che sia di tipo bigin con una lunghezza pari a 20, e che sia una chiave unica.</p>

31 <p>profileView verr&agrave; chiamato dal modulo userGroups ogni singola volta che dovr&agrave; mostrare la view di un profilo.<br/>

43 <h3>Creare views UserGroupsUser::VIEW</h3>
44 <p>Le pagine VIEW vengono caricate quando qualcuno carica un profilo<br/>
45 Nel tuo view file avrai la variabile <b>$model</b>, la quale conterr&agrave; il modello della tua Profile Extension.</p>

47 <h3>Creare view UserGroupsUser::UPDATE</h3>
48 <p>Nella tua update views avrai sempre il modello della tua Profile Extension all'interno della variabile <b>$model</b>.<br/>
49 Avrai inoltre una variabile nominata <b>$user_id</b>, la quale conterr&agrave; l'ID del profilo di cui si sta facendo l'editing.</p>
50 <p>Nella tua update view potrai utilizzare la validazione ajax e noi ti incoraggiamo a farlo, ma dovrai prestare particolare attenzione al bottone di invio.<br/>
51 Qui puoi vedere un esempio del bottone:</p>
52 
53 <div class="code"><code>
54  echo CHtml::ajaxSubmitButton(
55  Yii::t('userGroupsModule.general','Update External Profile'), // prima riga
56  Yii::app()-&gt;baseUrl . '/userGroups/user/update/id/'.$user_id, // seconda riga
57  array('update' =&gt; '#userGroups-container'), // terza riga
58  array('id' =&gt; 'submit-profile-'.$model-&gt;id.rand()) // quarta riga
59  );
60 
61 </code></div>

63 <p>Nella prima riga stiamo usando il metodo Yii::t() per definire il nome del botton. Non hai bisogno di fare altrettanto
64 se non ti interessa la localizzazione, pertanto puoi utilizzare una stringa.<br/>
65 Nella seconda riga settiamo il path del form. Assicurati di copiarlo cos&igrave; com'&egrave; almeno che tu non voglia
66 usare un tuo controller per gestire l'azione di salvataggio del form. In quel caso punta il form verso il path corretto.<br/>
67 Nella terza riga settiamo l'azione che il javascript eseguir&agrave; subito dopo l'esecuzione del form. Copiala esattamente
68 cos&igrave; com'&egrave;.<br/>
69 Nella quarta riga settiamo un nome casuale per l'id del bottone. Questa &egrave; una cosa importante da fare se non vuoi
70 avere problemi di overloading con ajax.</p>

72 <h3>Creare view UserGroupsUser::REGISTRATION</h3>
73 <p>Nei file delle view di registrazione dovrai semplicemente inserire i campi input del tuo form, e nient'altro.<br/>
74 I tuoi campi input saranno parte del form di registrazione e userGropus si occuper&agrave; della validazione e di salvare i dati sul database.<br/>
75 In queste view avrai il modello della tua Profile Extension all'interno della variabile <b>$model</b>.</p>

77 <h3>Quindi posso usare la normale validazione nel mio modello e tutto il resto?</h3>
78 <p>Certo che puoi, e potrai inoltre utilizzare tutte le altre feature dei modelli a cui sei abituato, come afterFind, beforeSave eccetera eccetera.<br/>
79 Al momento userGroups supporta le Profile Extensions solo per le azioni di view ed editing, non per la registrazione o il recovery.<br/>
80 Ad ogni modo ti incoraggiamo ad utilizzare regole basate sugli scenari per evitare futuri problemi.<br/>
81 Lo scenario utilizzato nelle view UserGroupsUser::UPDATE &egrave;: <b>updateProfile</b>.<br/>
82 Lo scenario utilizzato nelle view UserGroupsUser::REGISTRATION &egrave;: <b>registration</b>.</p>

98 <h3>Le Profile Extension utilizzando il sistema di relazioni di CActiveRecord?</h3>
99 <p>Ovviamente s&igrave;, userGroups crea al momento relazioni di tipo HAS_ONE con i modelli che tu inserisci, pertanto ogni volta che
100 vorrai usare il modello UserGroupsUser tutti i dati delle tue Profile Extensions verranno caricati.<br/>
101 Potrai ottenerli usando questa sintassi:
102 <b>$model-&gt;rel[model class name]</b><br/>
103 Pertanto considerando ancora l'esempio del model UserHobbies, la sintassi sarebbe la seguente.
104 <b>$model-&gt;relUserHobbies</b>
